Mobil uygulama için authentication servisleri

Genel Bakış

MindCoach API, mobil uygulama için authentication servisleri sağlar. Google, Facebook ve Apple ile OAuth authentication destekler.

Base URL: http://localhost:3000
🔐 Authentication Yöntemi: Stateful JWT
  • Token'lar database'de saklanır (user_tokens tablosu)
  • Her istekte token hem JWT signature ile verify edilir hem de database'de kontrol edilir
  • Logout işlemi yapılabilir (token revoke edilebilir)
  • Tüm cihazlardan logout yapılabilir
📋 İki Aşamalı Kayıt Süreci:
  • 1. Aşama: Google/Facebook/Apple ile ilk oturum açma - Sadece temel bilgiler (credential, credentialData) kaydedilir
  • 2. Aşama: Profil tamamlama - Kullanıcı soruları cevapladıktan sonra username, answerData ve diğer bilgiler eklenir

Facebook Authentication Kurulumu

📋 Geliştiricinin Yapması Gerekenler:
  1. Facebook Developer Console'da Uygulama Oluşturma:
    • Facebook Developer Console'a gidin
    • "My Apps" > "Create App" butonuna tıklayın
    • App türü olarak "Consumer" seçin
    • App adını ve contact email'i girin
    • App ID ve App Secret'ı kaydedin
  2. iOS Platform Ekleme:
    • Facebook Developer Console > Settings > Basic
    • "Add Platform" > "iOS" seçin
    • Bundle ID'yi girin (örn: com.example.mindcoach)
    • App Store ID'yi girin (opsiyonel)
  3. Android Platform Ekleme:
    • Facebook Developer Console > Settings > Basic
    • "Add Platform" > "Android" seçin
    • Package Name'i girin (örn: com.example.mindcoach)
    • Class Name'i girin (örn: com.example.mindcoach.MainActivity)
    • Key Hashes Ekleme (ÖNEMLİ):
      • Debug Key Hash: keytool -exportcert -alias androiddebugkey -keystore ~/.android/debug.keystore | openssl sha1 -binary | openssl base64
      • Release Key Hash: Production keystore'unuzdan aynı komutu çalıştırın
      • Key Hash'leri Facebook Developer Console'a ekleyin
  4. Facebook Login Özelliğini Aktifleştirme:
    • Facebook Developer Console > Products > Facebook Login > Settings
    • "Valid OAuth Redirect URIs" ekleyin (iOS ve Android için)
    • iOS için: fb{APP_ID}://authorize
    • Android için: fb{APP_ID}://authorize
  5. App Review (Production için):
    • Development mode'da sadece uygulama geliştiricileri giriş yapabilir
    • Production'da kullanmak için Facebook App Review'a başvurun
    • Gerekli izinler: email, public_profile
  6. Flutter Projesinde Yapılandırma:
    • iOS (Info.plist):
      • FacebookAppID: App ID'nizi ekleyin
      • FacebookClientToken: Client Token'ı ekleyin
      • FacebookDisplayName: Uygulama adınızı ekleyin
      • CFBundleURLSchemes: fb{APP_ID} formatında ekleyin
    • Android (strings.xml):
      • facebook_app_id: App ID'nizi ekleyin
      • facebook_client_token: Client Token'ı ekleyin
    • Android (AndroidManifest.xml):
      • Meta-data tag'lerini ekleyin (zaten eklenmiş)
  7. API Environment Variables:
    • FACEBOOK_APP_ID: Facebook App ID'nizi ekleyin
    • FACEBOOK_APP_SECRET: Facebook App Secret'ınızı ekleyin
    • Bu değişkenleri .env dosyanıza ekleyin
💡 Önemli Notlar:
  • Key Hash'ler Android için kritik öneme sahiptir. Eksik veya yanlış key hash, "Feature Unavailable" hatasına neden olur.
  • Development mode'da sadece uygulama geliştiricileri giriş yapabilir. Production'da kullanmak için App Review gerekir.
  • iOS için Bundle ID ve Android için Package Name'in Facebook Developer Console'daki değerlerle tam olarak eşleşmesi gerekir.
  • Client Token'ı Facebook Developer Console > Settings > Basic > Client Token'dan alabilirsiniz.

Apple Sign In Kurulumu

📋 Geliştiricinin Yapması Gerekenler:
  1. Apple Developer Console'da App ID Yapılandırması:
    • Apple Developer Console'a gidin
    • Certificates, Identifiers & Profiles > Identifiers
    • App ID'nizi seçin veya yeni bir App ID oluşturun
    • "Sign In with Apple" capability'sini aktifleştirin
    • App ID'yi kaydedin
  2. Service ID Oluşturma (Web ve Android için):
    • Identifiers > Services > "+" butonuna tıklayın
    • Service ID oluşturun (örn: com.example.mindcoach.service)
    • "Sign In with Apple" capability'sini aktifleştirin
    • Configure butonuna tıklayın
    • Primary App ID'yi seçin
    • Return URLs ekleyin (web için gerekli, Android için opsiyonel)
    • Service ID'yi kaydedin
  3. iOS Xcode Yapılandırması:
    • Xcode'da projeyi açın
    • Runner target'ını seçin
    • Signing & Capabilities sekmesine gidin
    • "+ Capability" butonuna tıklayın
    • "Sign In with Apple" capability'sini ekleyin
    • Veya Runner.entitlements dosyasına manuel olarak ekleyin: 
      <key>com.apple.developer.applesignin</key>
<array>
    <string>Default</string>
</array>
  4. Android Yapılandırması:
    • Apple Sign In Android'de de çalışır (sign_in_with_apple paketi destekler)
    • Ekstra yapılandırma gerekmez
    • Service ID kullanılır (iOS App ID yerine)
  5. API Environment Variables:
    • APPLE_CLIENT_ID veya APPLE_SERVICE_ID: Service ID'nizi ekleyin
    • Bu değişkeni .env dosyanıza ekleyin
    • Token verification sırasında audience olarak kullanılır
  6. Apple Politikalarına Uygunluk (ÖNEMLİ):
    • Hesap Silme:
      • Kullanıcılar hesabını silebilir (zaten implement edildi: DELETE /auth/account)
      • Hesap silme işlemi geri alınamaz olmalı (zaten öyle)
      • Tüm kullanıcı verileri kalıcı olarak silinmeli (zaten yapılıyor)
      • Privacy Policy ve Terms of Service linkleri uygulamada görünür olmalı (zaten var: assets/policies/)
    • Privacy Policy ve Terms of Service:
      • Uygulama içinde erişilebilir olmalı
      • App Store'da da belirtilmelidir
      • Mevcut dosyalar: assets/policies/en/, assets/policies/tr/, assets/policies/de/
    • Data Deletion:
      • Kullanıcı verileri silindiğinde, Apple'ın Account Deletion API'sine bildirim yapılabilir (opsiyonel)
      • Şu an için manuel silme yeterli (Apple'ın zorunlu kıldığı bir şey değil)
💡 Önemli Notlar:
  • Apple Sign In hem iOS hem de Android'de çalışır (sign_in_with_apple paketi destekler).
  • Email ve name bilgileri sadece ilk login'de sağlanır. Sonraki login'lerde sadece userIdentifier döner.
  • Identity token JWT formatındadır ve Apple'ın public key'leri ile verify edilir.
  • Token verification başarısız olsa bile, userIdentifier ile kullanıcı tanımlanabilir (fallback mekanizması).
  • Hesap silme işlemi Apple politikalarına uygundur: geri alınamaz, tüm veriler silinir.

JWT Token Mantığı

MindCoach API, kullanıcı authentication için Stateful JWT (Database'de token saklama) kullanır. Token'lar hem JWT signature ile verify edilir hem de database'de kontrol edilir. Bu bölümde token'ın nasıl çalıştığını detaylı olarak açıklıyoruz.

🔐 Stateful JWT Kullanımı:
  • Token'lar database'de saklanır (user_tokens tablosu)
  • Her istekte token hem JWT signature ile verify edilir hem de database'de kontrol edilir
  • Token revoke edilebilir (logout işlemi)
  • Tüm cihazlardan logout yapılabilir

JWT Token Nedir?

JWT (JSON Web Token), kullanıcı authentication için kullanılan güvenli bir token formatıdır. Token, kullanıcının kimliğini doğrulamak ve yetkilendirmek için kullanılır.

Token Nasıl Oluşturulur?

Kullanıcı ilk kez Google/Facebook/Apple ile oturum açtığında, API otomatik olarak bir JWT token oluşturur ve hem response'da döner hem de database'e kaydedilir (Stateful JWT).

1

Token Oluşturma Süreci

Ne Zaman?

  • POST /auth/google - Google ile oturum açıldığında
  • POST /auth/facebook - Facebook ile oturum açıldığında
  • POST /auth/apple - Apple ile oturum açıldığında

Token İçeriği:

Token Payload 
{
  "userId": 1,
  "iat": 1234567890,
  "exp": 1235172690,
  "iss": "mindcoach-api",
  "aud": "mindcoach-app"
}

Token Formatı

JWT token 3 bölümden oluşur:

Token Yapısı 
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VySWQiOjEsImlhdCI6MTIzNDU2Nzg5MCwiZXhwIjoxMjM1MTcyNjkwfQ.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
│─────────────────││──────────────────────────────────────────────────────││──────────────────────────────────────────│
      Header                    Payload                                              Signature
Token Bölümleri:
  • Header: Token tipi ve algoritma bilgisi
  • Payload: Kullanıcı bilgileri (userId, iat, exp, vb.)
  • Signature: Token'ın doğruluğunu kontrol eden imza

Token Nasıl Kullanılır?

Token oluşturulduktan sonra, her API isteğinde Authorization header'ında gönderilmelidir.

Request Header 
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Token Kullanıldığı Endpoint'ler:
  • PUT /auth/profile - Profil güncelleme
  • GET /auth/me - Kullanıcı bilgilerini alma
  • GET /auth/verify - Token doğrulama

Token Doğrulama Süreci (Stateful JWT)

Her istekte token şu adımlarla doğrulanır:

1

Token Format Kontrolü

Token geçerli bir JWT formatında mı?

2

Signature Kontrolü

Token imzası doğru mu? (JWT_SECRET ile kontrol edilir)

3

Expiration Kontrolü

Token süresi dolmuş mu? (Varsayılan: 7 gün)

4

Database Kontrolü (Stateful JWT)

Token database'de var mı ve revoke edilmemiş mi? (user_tokens tablosunda kontrol edilir)

5

User Kontrolü

Token'daki userId'ye sahip kullanıcı database'de var mı?

Token Özellikleri

⏰ Token Expiration (Süre Sınırı)

  • Varsayılan Süre: 7 gün
  • Token oluşturulduktan 7 gün sonra geçersiz olur
  • Kullanıcı yeniden oturum açmalıdır
  • Süre .env dosyasında JWT_EXPIRES_IN=7d ile ayarlanabilir

Token İşlem Akışı

1. İlk Oturum Açma

Kullanıcı → POST /auth/google { idToken }
           ↓
        Server token verify eder
           ↓
        Database'de kullanıcı kontrol edilir/oluşturulur
           ↓
        JWT token oluşturulur (userId ile)
           ↓
        Token kullanıcıya döner

2. Token ile İstek Yapma

Kullanıcı → GET /auth/me
           Authorization: Bearer <token>
           ↓
        Server token'ı verify eder
           ↓
        Token geçerli mi? (signature, expiration, database, user)
           ↓
        Database'de token kontrol edilir (revoked mı?)
           ↓
        Kullanıcı bilgileri request'e eklenir
           ↓
        İstek işlenir ve response döner

3. Token Süresi Dolduğunda

Kullanıcı → GET /auth/me
           Authorization: Bearer <expired_token>
           ↓
        Server token'ı verify eder
           ↓
        Token süresi dolmuş!
           ↓
        401 Unauthorized hatası döner
           ↓
        Kullanıcı yeniden oturum açmalı
⚠️ Token Süresi Dolduğunda Ne Yapılmalı?

Token süresi dolduğunda (7 gün sonra) kullanıcı yeniden oturum açmalıdır. API'de token refresh mekanizması yoktur.

Flutter'da Token Kullanımı

1. Token Kaydetme

Dart 
import 'package:flutter_secure_storage/flutter_secure_storage.dart';

final storage = FlutterSecureStorage();

Future<void> saveToken(String token) async {
  await storage.write(key: 'jwt_token', value: token);
}

2. Token Okuma

Dart 
Future<String?> getToken() async {
  return await storage.read(key: 'jwt_token');
}

3. Token ile İstek Yapma

Dart 
import 'package:http/http.dart' as http;
import 'dart:convert';

Future<Map<String, dynamic>> getUserProfile() async {
  final token = await getToken();
  
  if (token == null) {
    throw Exception('No token found. Please login again.');
  }

  final response = await http.get(
    Uri.parse('http://your-api-url/auth/me'),
    headers: {
      'Authorization': 'Bearer $token',
      'Content-Type': 'application/json',
    },
  );

  if (response.statusCode == 200) {
    return jsonDecode(response.body);
  } else if (response.statusCode == 401) {
    // Token geçersiz veya süresi dolmuş
    await storage.delete(key: 'jwt_token');
    throw Exception('Token expired. Please login again.');
  } else {
    throw Exception('Failed to get user profile');
  }
}

    final token = await storage.read(key: 'jwt_token');

      throw Exception('No token found');
    }

    final headers = {

      'Content-Type': 'application/json',

          Uri.parse('$baseUrl$endpoint'),

        throw TokenExpiredException('Token has expired');

Token Yönetimi Nasıl Çalışır?

1

Token Oluşturulduğunda

Kullanıcı oturum açtığında (Google/Facebook/Apple), JWT token oluşturulur ve hem response'da döner hem de zorunlu olarak user_tokens tablosuna kaydedilir. Token kaydetme başarısız olursa authentication başarısız olur.

2

Token Verify Edildiğinde

Her authenticated istekte, token şu sırayla kontrol edilir:

  1. JWT signature verify edilir (JWT_SECRET ile)
  2. Token expiration kontrol edilir
  3. Database'de token kontrol edilir (var mı, revoked mı?)
  4. User database'de var mı kontrol edilir

Önemli: Token database'de yoksa veya revoked ise, istek 401 hatası ile reddedilir.

3

Logout Yapıldığında

Logout endpoint'i çağrıldığında, token'ın is_revoked flag'i true olarak işaretlenir ve revoked_at timestamp'i set edilir. Token artık geçersizdir ve kullanılamaz.

4

Revoked Token Kontrolü

Token verify edilirken, database'de token'ın is_revoked flag'i kontrol edilir. Eğer true ise veya token database'de yoksa, token geçersiz kabul edilir ve 401 hatası döner.

⚠️ Önemli Notlar:
  • Stateful JWT Zorunlu: MindCoach API Stateful JWT kullanıyor. Token'lar database'de saklanmalı ve her istekte kontrol edilmelidir.
  • Database Migration Gerekli: Token yönetimi için user_tokens tablosu oluşturulmalıdır. Migration dosyası: database/migrations/003_create_user_tokens_table.sql
  • Performance: Token'ları database'de saklamak her istekte bir database sorgusu gerektirir, bu da performansı etkileyebilir. Ancak güvenlik ve yönetim için gerekli.
  • Security: Token'ları database'de saklamak, logout ve token yönetimi için daha güvenlidir. Revoked token'lar hemen geçersiz hale gelir.
  • Token Kaydetme Zorunlu: Token kaydetme başarısız olursa authentication başarısız olur. Token'lar mutlaka database'e kaydedilmelidir.

    Uri.parse('http://your-api-url/consultants/date/$createdDate'),
    headers: {'Content-Type': 'application/json'},
  );

  if (response.statusCode == 200) {
    final data = jsonDecode(response.body);
    final consultants = (data['data']['consultants'] as List)

Moods Table Schema 
CREATE TABLE IF NOT EXISTS `moods` (
  `id` INT AUTO_INCREMENT PRIMARY KEY,
  `user_id` INT NOT NULL COMMENT 'User ID - Kullanıcı ID',
  `date` DATE NOT NULL COMMENT 'Veri giriş tarihi',
  `mood` INT NOT NULL COMMENT 'Kullanıcının günlük modu (integer)',
  `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  `updated_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (`user_id`) REFERENCES `users`(`id`) ON DELETE CASCADE,
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_date` (`date`),
  INDEX `idx_user_date` (`user_id`, `date`),
  UNIQUE KEY `unique_user_date` (`user_id`, `date`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
COMMENT='Moods table - Kullanıcıların günlük modları';

Realtime Chat API

WebSocket (Socket.IO) kullanarak kullanıcıların koçlarla gerçek zamanlı sesli konuşmasını sağlayan API. Kullanıcılar anlık olarak sesli konuşur (streaming audio), sistem sesi metne çevirir, AI yanıt verir ve ElevenLabs ile sesli yanıt oluşturulur. 3D animasyon tetikleme desteği içerir.

🔌 WebSocket Bağlantısı:
  • Protocol: WebSocket (Socket.IO)
  • URL: ws://your-api-url veya wss://your-api-url (production)
  • Authentication: JWT token ile (handshake sırasında)
  • Transport: WebSocket ve Polling (fallback)
🎤 Anlık Sesli Konuşma Akışı:
  • 1. Konuşma Başlatma: Kullanıcı konuşmaya başladığında audio_stream_start event'i gönderilir
  • 2. Ses Chunk'ları: Kullanıcı konuşurken ses chunk'ları gerçek zamanlı olarak audio_stream_chunk ile gönderilir
  • 3. Konuşma Bitirme: Kullanıcı konuşmayı bitirdiğinde audio_stream_end event'i gönderilir
  • 4. Speech-to-Text: Tüm chunk'lar birleştirilir ve metne çevrilir (OpenAI Whisper veya webhook)
  • 5. AI İşleme: Transkript edilmiş metin AI asistanına gönderilir
  • 6. Koç Yanıtı: AI'dan gelen yanıt ElevenLabs ile sese dönüştürülür
  • 7. 3D Animasyon: Koç konuştuğunda animasyon tetiklenir
🎙️ ElevenLabs Entegrasyonu:
  • Her koçun kendine özel voice_id değeri vardır (consultants tablosunda)
  • Koç yanıtları otomatik olarak ses dosyasına (MP3) dönüştürülür
  • Ses dosyası base64 formatında gönderilir
  • Environment variable: ELEVENLABS_API_KEY gerekli
🗣️ Speech-to-Text Entegrasyonu:
  • OpenAI Whisper API kullanılabilir (OPENAI_API_KEY gerekli)
  • Alternatif olarak webhook servisi kullanılabilir (SPEECH_TO_TEXT_WEBHOOK_URL)
  • Kullanıcının native language'ı otomatik olarak kullanılır
  • Desteklenen formatlar: MP3, WAV, M4A, AAC, OGG
🎬 3D Animasyon Tetikleme:
  • Koç konuştuğunda animation_trigger event'i gönderilir
  • Event tipi: speak
  • Consultant ID ve timestamp bilgisi içerir
  • Flutter uygulaması bu event'i dinleyerek 3D animasyonu tetikler

Realtime Phone Call API

Gerçek zamanlı telefon görüşmesi gibi AI konuşması sağlayan API. Kullanıcılar mikrofon üzerinden anlık konuşur, AI anında yanıt verir. Barge-in (kesinti) desteği ile kullanıcı AI konuşurken konuşabilir ve AI derhal durur.

📞 Telefon Görüşmesi Benzeri Deneyim:
  • Gerçek Zamanlı: Kullanıcı konuşurken AI anında işler
  • Barge-in: AI konuşurken kullanıcı konuşursa AI derhal durur
  • Düşük Gecikme: Streaming audio ile minimum gecikme
  • Doğal Konuşma: Telefon görüşmesi gibi akıcı diyalog
🔌 WebSocket Bağlantısı:
  • Protocol: WebSocket (ws library)
  • URL: ws://your-api-url:3001 veya wss://your-api-url:3001 (production)
  • Port: 3001 (varsayılan, REALTIME_WS_PORT ile değiştirilebilir)
  • Authentication: JWT token query parameter ile
  • Format: Binary audio chunks (PCM 16kHz)
🔄 Sistem Akışı:
  • 1. Bağlantı: WebSocket bağlantısı kurulur (token + consultantId)
  • 2. Mikrofon Akışı: Flutter'dan PCM audio chunk'ları gerçek zamanlı gönderilir
  • 3. OpenAI Realtime: Audio chunk'ları OpenAI Realtime API'ye stream edilir
  • 4. Text Delta: OpenAI text delta'ları gerçek zamanlı gelir
  • 5. ElevenLabs TTS: Text delta'ları ElevenLabs Streaming TTS'ye gönderilir
  • 6. Audio Stream: ElevenLabs'tan gelen PCM audio Flutter'a stream edilir
  • 7. Barge-in: Kullanıcı konuşursa AI derhal durur
🎯 Barge-in (Kesinti) Mantığı:
  • AI konuşurken kullanıcıdan audio gelirse:
  • → ElevenLabs stream AbortController ile derhal durdurulur
  • → OpenAI Realtime response response.cancel ile iptal edilir
  • → AI konuşması tamamen kesilir
  • → Kullanıcı konuşması işlenmeye başlar
🔇 Silence Detection:
  • RMS (Root Mean Square) threshold kullanılır
  • Varsayılan threshold: 0.005
  • 1 saniye sessizlik sonrası AI yanıt oluşturmaya başlar
  • Silence detection gerçek zamanlı çalışır

    );

    channel = WebSocketChannel.connect(uri);

    channel!.stream.listen(

          // Binary audio data from ElevenLabs
          playAudioChunk(data);
        } else if (data is String) {
          // JSON message
          handleMessage(data);
        }
      },

      }

import 'package:http/http.dart' as http;

Future<Map<String, dynamic>> signInWithGoogle(String idToken) async {
  final response = await http.post(
    Uri.parse('http://your-api-url/auth/google'),
    headers: {'Content-Type': 'application/json'},
    body: jsonEncode({'idToken': idToken}),
  );

  if (response.statusCode == 200) {
    final data = jsonDecode(response.body);
    return {
      'user': UserModel.fromMap(data['data']['user']),
      'token': data['data']['token'],
    };
  } else {
    throw Exception('Authentication failed');
  }
}

  }
}
